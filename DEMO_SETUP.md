# Demo Setup Guide for New Client Demos

This guide explains how to customize the DLX Airline Demo for a new customer presentation.

## Quick Reference: Files to Change

| What to Change | File(s) | Line(s) |
|----------------|---------|---------|
| Gr4vy Instance ID | `src/app/api/token/route.js` | Line 21 |
| Gr4vy Instance ID (frontend) | `src/app/checkout/CheckoutContent.js` | Line 267 |
| Merchant Account ID | `src/app/api/token/route.js` | Line 24 |
| Private Key | `.env.local` | `GR4VY_PRIVATE_KEY=...` |
| Environment (sandbox/production) | `src/app/api/token/route.js` + `CheckoutContent.js` | Lines 23 & 268 |
| Logo/Branding | Multiple page files | See "UX Branding" section |
| App Title | `src/app/layout.js` | Line 15 |

## 1. Gr4vy Configuration (Required)

### Instance ID (`gr4vyId`)

You need to update the instance ID in **two places**:

#### Backend - `src/app/api/token/route.js`
```js
// Line 21: Change 'datalex' to the client's instance ID
const client = new Client({
  gr4vyId: 'datalex',  // ← CHANGE THIS
  privateKey: privateKey,
  environment: 'sandbox',
  merchantAccountId: 'default'
});
```

#### Frontend - `src/app/checkout/CheckoutContent.js`
```js
// Line 267: Change 'datalex' to the client's instance ID
<Embed
  ref={embedRef}
  gr4vyId="datalex"  // ← CHANGE THIS
  environment="sandbox"
  ...
/>
```

### Merchant Account ID

Update in `src/app/api/token/route.js`:
```js
// Line 24: Change 'default' to client's merchant account ID if needed
merchantAccountId: 'default'  // ← CHANGE THIS if needed
```

### Private Key (`.env.local`)

Create or update the `.env.local` file in the project root:

```bash
GR4VY_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----
YOUR_PRIVATE_KEY_CONTENT_HERE
-----END PRIVATE KEY-----
```

**Note:** If your private key has literal `\n` characters, the app will handle them automatically (see line 18 in route.js).

### Environment (Sandbox vs Production)

The environment is currently set to `sandbox`. Update in **two places** if switching to production:

1. `src/app/api/token/route.js` - Line 23: `environment: 'sandbox'`
2. `src/app/checkout/CheckoutContent.js` - Line 268: `environment="sandbox"`

## 2. UX/Branding Changes

### Logo

The logo appears as an SVG in multiple files. Search for the Datalex logo (the "X" cross pattern) and replace with the client's logo.

| Page | File | Look For |
|------|------|----------|
| Home | `src/app/page.js` | SVG starting around line 47 |
| Flights | `src/app/flights/page.js` | SVG starting around line 281 |
| Checkout | `src/app/checkout/CheckoutContent.js` | SVG starting around line 128 |

**Tip:** The current logo is a Datalex SVG. Replace the entire `<svg>...</svg>` block with the new client's logo. You can:
- Use an SVG directly in the code
- Replace with an `<img>` tag pointing to `/public/logo.png`

### Tagline Text

The "Airline" tagline under the logo can be found in:
- `src/app/page.js` - Line 62-64

```js
<p style={{ color: '#1a237e', marginTop: '12px', fontSize: '16px', fontWeight: '500' }}>
  Airline  // ← Change this text
</p>
```

### App Title (Browser Tab)

Update in `src/app/layout.js`:
```js
// Line 14-17
export const metadata = {
  title: "Create Next App",  // ← CHANGE THIS
  description: "Generated by create next app",  // ← CHANGE THIS
};
```

### Color Scheme

The primary color scheme is defined inline. The main colors used are:

| Color | Hex | Usage |
|-------|-----|-------|
| Primary Dark Blue | `#1a237e` | Headers, text, borders |
| Secondary Blue | `#283593` | Gradients |
| Accent Gradient | `#667eea` → `#764ba2` | Buttons, highlights |

To change colors globally, search and replace these hex values across:
- `src/app/page.js`
- `src/app/flights/page.js`
- `src/app/addons/page.js`
- `src/app/checkout/CheckoutContent.js`
- `src/app/success/page.js`

### "Powered by" Footer

The DPLX logo in the footer appears in:
- `src/app/page.js` (around lines 207-219)
- `src/app/success/page.js` (around lines 160-172)

You may want to remove or update these references.

## 3. Flight & Market Configuration

### Flight IDs

Flight IDs like `DLX101`, `DLX102` etc. are defined in `src/app/flights/page.js`:

```js
// Lines 68-73
const flights = [
  { id: 'DLX101', departure: '08:00', arrival: '11:30', duration: '3h 30m' },
  { id: 'DLX102', departure: '12:00', arrival: '15:30', duration: '3h 30m' },
  // ... change 'DLX' prefix to client's airline code
];
```

### Market/Currency Support

Markets and currencies are defined in multiple files. The main mapping is:

```js
const currencies = {
  IE: { symbol: '€', code: 'EUR' },
  GB: { symbol: '£', code: 'GBP' },
  US: { symbol: '$', code: 'USD' },
  BR: { symbol: 'R$', code: 'BRL' },
  AE: { symbol: 'د.إ', code: 'AED' },
  RO: { symbol: 'lei', code: 'RON' }
};
```

To add/remove markets, update in:
- `src/app/page.js` - Dropdown options (lines 99-106)
- `src/app/flights/page.js` - Currency mapping & price multipliers
- `src/app/addons/page.js` - Currency mapping
- `src/app/checkout/CheckoutContent.js` - Currency mapping & country codes
- `src/app/success/page.js` - Currency mapping

## 4. Running the Demo

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
npm start
```

Open [http://localhost:3000](http://localhost:3000) to see the demo.
